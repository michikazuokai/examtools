今回の修正の目標は、次のように整理するとよいです。

# 今回の修正の到達目標

```text
試験問題生成システムを、subjectだけで各工程を個別実行できる構成に整理する。

そのために、プログラム名・実行引数・入力取得・Excel読み込み・hash/stamp確認・ログ出力を統一し、
古いExcel・古いJSON・古い中間ファイルを誤って使わない運用にする。

一括実行スクリプトや数珠つなぎ実行は、今回の対象外とする。
```

------

# 1. プログラム名を統一する

現在のスクリプト名を、役割が分かる名前に整理する。

```text
validate_excel.py   試験問題.xlsxをチェック・補正する
make_json.py        JSONを作成する
make_latex.py       LaTeX本文を作成する
make_pdf.py         PDFを作成する
make_anspdf.py      解答用紙PDFを作成する
```

実行順は次の通り。

```text
1. validate_excel.py
2. make_json.py
3. make_latex.py
4. make_pdf.py
5. make_anspdf.py
```

------

# 2. 実行引数を統一する

基本の実行形式は、すべて **subjectのみ** に統一する。

```bash
python scripts/validate_excel.py 1020701
python scripts/make_json.py 1020701
python scripts/make_latex.py 1020701
python scripts/make_pdf.py 1020701
python scripts/make_anspdf.py 1020701
```

次の引数は原則として使わない。

```text
--fsyear   → 設定・環境側から取得
--sheet    → subject または環境側から決定
--version  → Excel内の qpattern から決定
```

確認用のオプションは、まず `validate_excel.py` のみに残す。

```bash
python scripts/validate_excel.py 1020701 --dryrun
```

`--dry-run` ではなく、`--dryrun` に統一する。

------

# 3. 処理の共通化を行う

各プログラムで重複している処理を `utils.py` に集約する。

共通化する処理は次の通り。

```text
・subject 引数の受け取り
・試験問題.xlsx の場所取得
・exam_dir の取得
・work_dir の取得
・sheetname の決定
・必要に応じた workbook / worksheet の読み込み
・qpattern の取得
・Excel hash の計算
・validation_stamp の読み書き
・JSON metainfo の読み取り
・JSON と Excel の hash 照合
・エラー表示の形式
・ログ出力の形式
```

各プログラムでは、できるだけ次のように共通処理を呼び出す形にする。

```python
exam_context = load_exam_context(subject, load_workbook=True)
```

`exam_context` には、次の情報を持たせる。

```text
exam_context.subject
exam_context.excel_path
exam_context.exam_dir
exam_context.work_dir
exam_context.sheetname
exam_context.workbook
exam_context.worksheet
```

`ctx` という短縮名ではなく、分かりやすく `exam_context` を使う方針にする。

------

# 4. validate_excel.py にチェックと補正を集約する

`validate_excel.py` は、`make_json.py` の前に必ず実行する前処理として位置づける。

実行内容は次の通り。

```text
・C列 qid を毎回自動セットする
・select / subselect のG列にB版シャッフル番号をセットする
・answer / subanswer のG列にB版正解番号をセットする
・タグ構造をチェックする
・orderB をチェックする
・点数状況を表示する
・必要なセルコメントをセットする
・必要なセルスタイルをセットする
・ログを出力する
・エラーがなければ validation_stamp を作成する
```

JSON contract 以外のExcel入力チェックは、できるだけ `validate_excel.py` に寄せる。

------

# 5. hash / stamp による整合性確認を行う

基準は常に、

```text
試験問題.xlsx の対象シート内容ハッシュ
```

とする。

`utils.calc_excel_hash(ws)` を共通で使用する。

各段階の役割は次の通り。

```text
validate_excel.py
  → validation_stamp に source_excel_hash を保存

make_json.py
  → validation_stamp と現在のExcel hashを照合
  → JSON metainfo に source_excel_hash を保存

make_latex.py
  → JSON metainfo の source_excel_hash を読み、LaTeXコメントに引き継ぐ

make_pdf.py
  → PDF作成ログまたは build_info に source_excel_hash を記録する

make_anspdf.py
  → JSONの source_excel_hash と現在のExcel hash を照合する
```

------

# 6. 古い中間ファイルの誤使用を防ぐ

次のような事故を防ぐ。

```text
・Excelを修正したのに、古いJSONからPDFを作ってしまう
・JSONを作り直していないのに、解答用紙を作ってしまう
・A/B版の設定変更後に古い中間ファイルを使ってしまう
```

特に重要なのは次の2つ。

```text
make_json.py
  → validate_excel.py 実行後のExcelかどうかを確認する

make_anspdf.py
  → 解答用紙作成時に、JSONと現在のExcelが一致しているか確認する
```

------

```text

・点数状況
```

# 8. 一括実行は今回の対象外にする

今回は、次は行わない。

```text
・一括実行スクリプトを作る
・validate_excel.py から subprocess で次工程を呼ぶ
・数珠つなぎ形式で自動実行する
・JSON辞書をメモリ上で次工程へ渡す
```

各プログラムは、あくまで個別に実行する。

```bash
python scripts/validate_excel.py 1020701
python scripts/make_json.py 1020701
python scripts/make_latex.py 1020701
python scripts/make_pdf.py 1020701
python scripts/make_anspdf.py 1020701
```

その代わり、各工程で hash / stamp を確認し、古い中間ファイルを使わないようにする。
