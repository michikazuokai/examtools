はい、**解答用紙作成時のチェックにも非常に有効**です。
むしろ、今回の stamp / hash の考え方は、`anstest_ans.py` の混乱防止にかなり向いています。

現在の混乱は、おそらく次の状態です。

```text
試験問題.xlsx は修正済み
しかし answers_1020701.json は古いExcelから作られたまま
↓
解答用紙を作ると、ExcelとJSONの内容が合わない
```

これを防ぐには、`answers_1020701.json` の中に、作成元Excelのハッシュを入れておき、`anstest_ans.py` 実行時に現在のExcelのハッシュと照合します。

## 運用イメージ

```text
validate_excel.py --fix
  ↓
試験問題.xlsx を整える
  ↓
validation_stamp に Excel hash を保存

maketexjson.py
  ↓
試験問題.xlsx から answers_1020701.json を作る
  ↓
JSON内の metainfo に同じ Excel hash を保存

anstest_ans.py
  ↓
現在の試験問題.xlsx の hash を計算
  ↓
answers_1020701.json の metainfo.hash と比較
  ↓
一致しなければ停止
```

## `anstest_ans.py` でできるチェック

解答用紙を作る前に、次を確認できます。

```text
1. answers_1020701.json が存在するか
2. JSON内に metainfo.hash があるか
3. 現在の試験問題.xlsx の hash と一致するか
4. 一致しなければ「JSONが古い」と判断して停止
```

エラーメッセージは、たとえば次のようにできます。

```text
answers_1020701.json は現在の試験問題.xlsx と一致しません。

試験問題.xlsx が更新された可能性があります。
先に maketexjson.py を再実行してください。

Excel hash : xxxxx
JSON hash  : yyyyy
```

これなら、「解答用紙が古いJSONを使っている」ことに気づけます。

## `calc_excel_hash(ws)` を共通で使える

はい、ここでも `utils.py` の

```python
calc_excel_hash(ws)
```

を使えばよいです。

`maketexjson.py` がJSONに保存する hash と、`anstest_ans.py` がExcelから再計算する hash を同じ関数で作れば、比較できます。

## 追加する関数イメージ

`utils.py` に次のような共通関数を置くとよいです。

```python
def assert_json_matches_excel(json_meta: dict, ws, *, label: str = "answers JSON") -> None:
    excel_hash = calc_excel_hash(ws)
    json_hash = json_meta.get("hash")

    if not json_hash:
        raise ValueError(f"{label} に hash 情報がありません。maketexjson.py を再実行してください。")

    if excel_hash != json_hash:
        raise ValueError(
            f"{label} は現在の試験問題.xlsx と一致しません。\n"
            f"試験問題.xlsx が更新された可能性があります。\n"
            f"先に maketexjson.py を再実行してください。\n"
            f"Excel hash: {excel_hash}\n"
            f"JSON hash : {json_hash}"
        )
```

`anstest_ans.py` 側では、

```python
cjinfo = jsonmetainfo(work_dir, f"answers_{subject}.json")
assert_json_matches_excel(cjinfo, ws, label=f"answers_{subject}.json")
```

のように使います。

## さらに安全にするなら

`hash` だけでなく、次もJSONの `metainfo` に入れておくと運用確認が楽です。

```json
{
  "metainfo": {
    "inputpath": "/.../17/試験問題.xlsx",
    "sheetname": "1020701",
    "hash": "xxxx",
    "created_at": "2026-05-02T17:10:00",
    "tool": "maketexjson.py"
  }
}
```

こうしておくと、`anstest_ans.py` で

```text
このJSONは、どのExcelから、いつ作られたか
```

も表示できます。

## 結論

はい、stamp / hash の考え方は **解答用紙作成時のチェックに使えます**。
特に、今困っている「ExcelとJSONのバージョン違い」を防ぐには、かなり効果があります。

短期的には、`anstest_ans.py` に

```text
現在のExcel hash と answers JSON の hash を比較する
```

チェックを入れるのがおすすめです。
