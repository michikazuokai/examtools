# **📘 仕様追加設計書：前提条件ブロック（premise）**

## **🎯 目的**

試験問題において、

- 試験全体の前提条件
- 大問間の共通条件
- 図や注意事項

を明示的に表示できる構造を追加する。

------

## **🧩 新規タグ仕様**

### **📌 タグ一覧**

```text
b_premise
b_preline
pretext
e_preline
preimage
e_premise
```

------

## **🧱 構造**

```text
b_exam
  b_premise
    b_preline
      pretext
      pretext
    e_preline
    preimage
  e_premise

  b_question
  ...
  e_question

  b_premise
  ...
  e_premise

  b_question
  ...
  e_question
e_exam
```

------

## **🧾 意味定義**

| **タグ**  | **意味**             |
| --------- | -------------------- |
| b_premise | 前提条件ブロック開始 |
| b_preline | テキスト複数行開始   |
| pretext   | 前提条件テキスト1行  |
| e_preline | テキスト終了         |
| preimage  | 前提条件画像         |
| e_premise | 前提条件ブロック終了 |

------

## **📦 JSON構造**

```json
{
  "type": "premise",
  "title": "前提条件",
  "content": [
    {
      "type": "preline",
      "values": [
        "この試験ではGitの基本操作を前提とします。",
        "コマンドは授業範囲内です。"
      ]
    },
    {
      "type": "image",
      "path": "gitflow.png",
      "width": 0.8
    }
  ]
}
```

------

## **🎨 LaTeX出力仕様**

```latex
\begin{premisebox}{前提条件}
本文（text / image）
\end{premisebox}
```

### **枠線仕様**

- 枠あり
- タイトルあり「前提条件」
- 通常本文より少し余白あり

------

## **⚠️ 改ページ仕様（重要）**

ここが今回の設計のポイントです。

### **✔ 原則**

```text
premise は単独ブロックとして扱う
```

------

### **✔ 改ページルール**

| **条件**                        | **動作**                 |
| ------------------------------- | ------------------------ |
| 前にPAGEBREAKあり               | そのまま改ページ         |
| premiseがページ末尾に収まらない | 自動で改ページ           |
| premiseの途中分割               | ❌禁止（1ブロックで扱う） |

------

### **✔ 実装方針（推奨）**

#### **方法①（簡単）**

```latex
\begin{samepage}
\begin{premisebox}
...
\end{premisebox}
\end{samepage}
```

👉 ブロック丸ごと保持

------

#### **方法②（より安全）**

```latex
\needspace{5\baselineskip}
```

👉 はみ出る前に改ページ

------

### **✔ 結論**

👉 **samepage or needspace を必ず入れる**

------

## **🚫 制約**

- b_question の中には入れない
- b_subgroup の中にも入れない
- 入れ子は1段階のみ
